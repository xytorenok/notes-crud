
# Техническое задание: Frontend-разработчик (NoteKeeper)

## 1. Общее описание
Необходимо разработать клиентскую часть приложения для ведения текстовых заметок. Приложение должно быть одностраничным (SPA) и работать на чистом JavaScript (Vanilla JS) без использования фреймворков.

## 2. Технологический стек
- HTML5, CSS3 (без Tailwind/Bootstrap).
- Vanilla JavaScript (ES6+).
- Fetch API для взаимодействия с бэкендом.

## 3. Функциональные требования
### Экран авторизации
- Форма с полями: Email, Пароль.
- Переключатель между режимами "Вход" и "Регистрация".
- Валидация: поля не должны быть пустыми.
- Обработка ошибок: вывод сообщений от API (например, "Неверный пароль").

### Главный экран (Заметки)
- Доступен только после успешного входа.
- Поле ввода текста заметки (textarea) и кнопка "Сохранить".
- Список заметок, отображающий текст и дату создания.
- Кнопка "Удалить" для каждой заметки.
- Кнопка "Выйти" (очистка сессии и возврат на экран входа).

## 4. Протокол взаимодействия с API
Бэкенд будет доступен по базовому адресу `/api`. Все запросы должны использовать `application/json`.

1. **Регистрация**: `POST /auth/register` -> Payload: `{email, password}`.
2. **Вход**: `POST /auth/login` -> Payload: `{email, password}`. Ответ: `{id, email}`.
3. **Получение заметок**: `GET /notes?userId={id}`. Ответ: массив объектов `[{_id, content, date}]`.
4. **Создание заметки**: `POST /notes` -> Payload: `{userId, content}`.
5. **Удаление**: `DELETE /notes/{id}`.

## 5. Требования к UI/UX
- Адаптивный дизайн (Mobile First).
- Состояние "Loading" при отправке форм.
- Сохранение данных пользователя (ID и Email) в `localStorage` для поддержания сессии при перезагрузке.
- Экранирование HTML-тегов в тексте заметок (защита от XSS).
