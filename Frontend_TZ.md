
# ТЗ: Frontend-разработчик (Модуль NoteKeeper Client)

## 1. Задача
Разработать клиентское SPA-приложение для ведения заметок на чистом JavaScript.

## 2. Структура проекта
Исполнитель должен предоставить файлы в папке `/public`:
- `/public/index.html` — основная разметка и внедренные CSS-стили (допускается отдельный `style.css`).
- `/public/index.js` — логика приложения (ES6 модули).
- `/public/assets/` — (опционально) иконки или шрифты.

## 3. Требования к реализации
- **Интерфейс**: Адаптивный (мобильные/десктоп). Состояния: "Авторизация" (Login/Register) и "Рабочее пространство".
- **Безопасность**: Обязательное экранирование пользовательского ввода перед рендерингом (защита от XSS).
- **Сетевое взаимодействие**: Использовать `fetch`. Все запросы к API должны идти по относительным путям (напр. `/api/notes`), так как фронтенд будет обслуживаться тем же сервером.
- **Хранение сессии**: После успешного логина сохранять `user_id` и `email` в `localStorage`.

## 4. API Contract (Ожидаемое поведение бэкенда)
Фронтенд ожидает, что сервер слушает следующие эндпоинты:
- `POST /api/auth/login` -> `{email, password}`
- `POST /api/auth/register` -> `{email, password}`
- `GET /api/notes?userId=...` -> `[{_id, content, date}]`
- `POST /api/notes` -> `{userId, content}`
- `DELETE /api/notes/{id}`

## 5. Визуальный стиль
Минимализм, шрифт Inter, цветовая схема: Indigo (#4f46e5) для кнопок, светлый фон (#f8fafc).
