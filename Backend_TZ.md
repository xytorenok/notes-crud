
# Техническое задание: Backend-разработчик (NoteKeeper)

## 1. Общее описание
Необходимо разработать REST API сервер на Node.js для обслуживания приложения заметок. Сервер должен обеспечивать хранение данных в MongoDB и предоставлять эндпоинты для авторизации и CRUD операций.

## 2. Технологический стек
- Node.js (предпочтительно встроенный модуль `http` или `express`).
- Драйвер `mongodb`.
- База данных: MongoDB (использовать БД с именем `notes_app`).

## 3. Модели данных
### User
- `email` (string, unique)
- `password` (string)

### Note
- `userId` (string/ObjectId)
- `content` (string)
- `date` (string/date)

## 4. API Эндпоинты
Все ответы должны быть в формате JSON.

### Авторизация
- `POST /api/auth/register`: Создает пользователя. Проверять на дубликаты email.
- `POST /api/auth/login`: Проверяет email/password. Возвращает объект пользователя (включая `id`) или 401.

### Заметки
- `GET /api/notes?userId={id}`: Возвращает список всех заметок конкретного пользователя, отсортированных по дате (новые сверху).
- `POST /api/notes`: Принимает `{userId, content}`. Автоматически добавляет текущую дату/время.
- `DELETE /api/notes/{id}`: Удаляет заметку по её уникальному идентификатору.

## 5. Технические требования
- Обработка ошибок: при ошибках БД возвращать статус 500 с описанием.
- CORS: разрешить запросы с фронтенда (если развернуты на разных доменах).
- Парсинг: корректная обработка JSON в теле запроса (Body).
- Порт: брать из переменной окружения `process.env.PORT` или 3000 по умолчанию.
- Подключение к БД: брать из `process.env.MONGODB_URI`.
