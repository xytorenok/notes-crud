
# ТЗ: Backend-разработчик (Модуль NoteKeeper API)

## 1. Задача
Разработать REST API сервер на Node.js с интеграцией MongoDB.

## 2. Структура проекта
Исполнитель предоставляет файлы в корне репозитория:
- `server.js` — основной файл сервера.
- `package.json` — зависимости (обязательно `mongodb`).
- `.env.example` — шаблон переменных окружения.

## 3. Технические требования
- **База данных**: MongoDB. Использовать БД с именем `notes_app`.
- **Сервер**: Нативный модуль `http` или библиотека `express`.
- **Статика**: Сервер **обязан** отдавать статические файлы (html, js, css) из папки `./public`.
- **Конфигурация**: Порт (`PORT`) и URI базы данных (`MONGODB_URI`) должны браться из `process.env`.

## 4. Спецификация API (Эндпоинты)
Все запросы и ответы должны использовать `Content-Type: application/json`.

### 4.1 Авторизация
- `POST /api/auth/register`: Принимает `{email, password}`. Проверяет, не занят ли email. Возвращает объект созданного пользователя или ошибку 400.
- `POST /api/auth/login`: Принимает `{email, password}`. Сверяет данные. Возвращает `{id, email}` при успехе или 401 при ошибке.

### 4.2 Заметки
- `GET /api/notes?userId={id}`: Возвращает массив объектов заметок пользователя. Сортировка: от новых к старым (`_id: -1`). Пример ответа: `[{ "_id": "...", "content": "текст", "date": "10.10.2023, 12:00" }]`.
- `POST /api/notes`: Принимает `{userId, content}`. Сохраняет заметку, автоматически добавляя поле `date` (текущая дата/время строкой).
- `DELETE /api/notes/{id}`: Удаляет заметку по её идентификатору.

## 5. Ожидаемая структура БД
- Коллекция `users`: `{ _id, email, password }`.
- Коллекция `notes`: `{ _id, userId, content, date }`.
